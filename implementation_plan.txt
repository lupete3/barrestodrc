# Plan d'Implémentation : RestoBKV POS App

L'objectif est de créer une application web de Point de Vente (POS) pour bars, restaurants et boîtes de nuit à Bukavu. L'application doit être extrêmement robuste, fonctionner hors-ligne (offline-first), et être adaptée aux écrans tactiles Android avec de grands boutons et une esthétique moderne haut de gamme.

## Architecture & Technologies
- **Frontend Core** : React avec Vite (très rapide, HMR instantané, temps de build minime).
- **Styling** : Vanilla CSS pour un contrôle total, en créant un design premium (glassmorphism subtil, mode sombre élégant pour les environnements de boîte de nuit, et interactions fluides).
- **Stockage Local (Offline)** : IndexedDB via localforage ou Zustand avec persist, pour stocker les menus, les commandes non synchronisées, et les sessions serveurs locale.
- **PWA (Progressive Web App)** : Configuration de `vite-plugin-pwa` avec Service Worker pour permettre l'installation sur les tablettes Android et le fonctionnement sans connexion réseau (cache des ressources statiques).
- **Backend / DB Online** : Pour la synchronisation avec le Cpanel/VPS (à ajouter ultérieurement ou via des endpoints d'API fictifs synchronisables).

## Expérience Utilisateur (UI/UX)
- **POS First (Serveurs)** : Interface sans distraction, navigation par "grosses catégories" (Boissons, Nourritures, Extras), grille de produits avec grandes zones interactives pour éviter les erreurs de saisie sous pression. Gestion du "Panier" (Order Cart) toujours visible sur le côté.
- **Section Admin / Gérant (Backoffice)** : Réservé aux managers. Tableaux de bord intuitifs (rapports de vente), paramétrage des serveurs, des produits, et gestion des dépenses.
- **Impression** : Clic rapide pour lancer l'impression des tickets/factures, formaté pour les imprimantes thermiques 80mm ou 58mm via CSS `@media print`.
